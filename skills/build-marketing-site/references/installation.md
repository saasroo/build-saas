# Installation Steps

## Prerequisites

1. **Hugo Extended** (v0.80.0 or later)
   - Must be the Extended edition (required for TailwindCSS processing)
   - Verify: `hugo version` -- output must include "extended"
   - Download: https://github.com/gohugoio/hugo/releases

2. **Node.js and npm** (v14.0.0 or later)
   - Required for TailwindCSS compilation
   - Verify: `node --version` and `npm --version`

3. **Git**
   - Required for theme installation as submodule
   - Verify: `git --version`

## New Site Installation

```bash
# Step 1: Create new Hugo site
hugo new site <site-name>
cd <site-name>

# Step 2: Initialize git
git init

# Step 3: Add theme as git submodule
git submodule add https://github.com/chaoming/hugo-saasify-theme.git themes/hugo-saasify-theme
git submodule update --init --recursive

# Step 4: Copy build config files to site root
cp themes/hugo-saasify-theme/package.json .
cp themes/hugo-saasify-theme/postcss.config.js .
cp themes/hugo-saasify-theme/tailwind.config.copy.js ./tailwind.config.js

# Step 5: Install npm dependencies
npm install

# Step 6: Create hugo.toml (replace the default one)
# The theme requires specific build settings -- see hugo-toml-template.md
```

## Existing Site Installation

```bash
# Step 1: Add theme as submodule (from site root)
git submodule add https://github.com/chaoming/hugo-saasify-theme.git themes/hugo-saasify-theme
git submodule update --init --recursive

# Step 2: Copy build config files
cp themes/hugo-saasify-theme/package.json .
cp themes/hugo-saasify-theme/postcss.config.js .
cp themes/hugo-saasify-theme/tailwind.config.copy.js ./tailwind.config.js

# Step 3: Install npm dependencies
npm install

# Step 4: Update hugo.toml -- set theme and add required build settings
# theme = "hugo-saasify-theme"
# [build] writeStats = true
# [module.hugoVersion] extended = true, min = "0.80.0"
```

## Directory Structure After Install

```
my-site/
├── archetypes/
├── assets/
├── content/             # Your site content (Markdown)
├── data/
├── i18n/                # Translation files (create if multilingual)
├── layouts/             # Override theme layouts here
├── node_modules/        # npm dependencies (gitignored)
├── static/
│   └── images/          # Your images
├── themes/
│   └── hugo-saasify-theme/  # The theme (git submodule)
├── hugo.toml            # Site configuration
├── package.json         # Copied from theme
├── package-lock.json    # Generated by npm install
├── postcss.config.js    # Copied from theme
└── tailwind.config.js   # Copied from theme (customizable)
```

## Running the Site

```bash
# Development (with TailwindCSS hot reload)
npm start
# Site available at http://localhost:1313

# Production build
npm run build
# Output in public/ directory
```

**Important**: Always use `npm start` instead of `hugo server`. The npm script runs TailwindCSS compilation alongside Hugo. Running `hugo server` directly will result in missing styles.

## Updating the Theme

```bash
git submodule update --remote themes/hugo-saasify-theme

# If package.json changed, re-copy and reinstall
cp themes/hugo-saasify-theme/package.json .
cp themes/hugo-saasify-theme/postcss.config.js .
cp themes/hugo-saasify-theme/tailwind.config.copy.js ./tailwind.config.js
npm install
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| CSS not loading | Ensure `writeStats = true` in hugo.toml `[build]` section |
| Theme not found | Check `theme = "hugo-saasify-theme"` in hugo.toml |
| Hugo Extended error | Download Hugo Extended edition from GitHub releases |
| Empty theme directory | Run `git submodule update --init --recursive` |
| npm install fails | Delete `node_modules/` and `package-lock.json`, then `npm install` |
| Port 1313 in use | Kill process: `lsof -ti:1313 \| xargs kill -9` then retry |
